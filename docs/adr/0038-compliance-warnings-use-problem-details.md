# Compliance Warnings Use Problem Details

The A2A Simulator logs likely A2A or Jira remote-agent non-compliance as non-blocking Compliance Warnings using the project's Problem Details fields: `type`, `title`, `status`, `detail`, `timestamp`, and optional `instance`. Semantic non-compliance warnings use status `422`; malformed scenario structure that prevents execution uses `400` Problem Details instead. The `type` value should point to a real public specification URL where possible, such as the A2A specification or Atlassian's Jira remote-agent guide. Coarse-grained warnings can point to the relevant specification page; fine-grained warnings should point to a specific public anchor when one exists. These warnings are log entries, not response errors or simulated A2A stream events, so semantically unusual scenarios can still run for client testing.
