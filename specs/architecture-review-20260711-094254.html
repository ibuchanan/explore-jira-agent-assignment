Problem

Transport interface knowledge leaks through server handlers and tests: SSE headers, JSON-RPC frames, error frames, mapped-event formatting, closing, and stream parsing.

Solution

Make one stream adapter module own A2A-visible event formatting and SSE JSON-RPC framing while lifecycle modules emit domain-shaped events.

Wins

  • locality: wire rules concentrate
  • leverage: one adapter, many handlers
  • tests cross same seam
  • transport stops leaking
Supports ADR 0022: transport stays separate from state mapping. The mapper still does not learn SSE.

Problem

runScenario is now production-unused playback policy, while the Session core owns cursor, delay, pause, resume, resubscribe, and cancel policy.

Solution

Keep the step-to-event implementation, but remove or rename the obsolete playback module so there is one place to understand Session execution.

Wins

  • deletion test passes
  • fewer stale tests
  • Session keeps depth
Aligns with ADR 0039: Session policy belongs in the sans-IO core.

Deepen message intake for Scenario Matching

Worth exploring in-process

Problem

A2A message parsing, context creation, history writes, task text extraction, and Scenario Matching input are split across private server helpers and the matcher still only sees text.

Solution

Move incoming message normalization into one module that produces context updates and deterministic Scenario Matching input.

Wins

  • locality: intake casts vanish
  • structured context unlocks
  • matcher interface shrinks
  • resume rule stays visible
Preserves ADR 0036: follow-up user input still resumes paused tasks; the module only makes that input explicit.

Problem

One shared module mixes A2A task and stream contract, JSON-RPC method params, demo advance shapes, installation/context storage guidance, validators, and formatters.

Solution

Keep the protocol contract deep, but move sample-storage guidance and demo-only shapes away from the contract interface.

Wins

  • interface stops widening
  • imports express intent
  • unknown state gets revisited
  • leverage for both apps
Supports ADR 0022. Watch the unknown mismatch: code treats it as active/transitionable while the domain model calls it exceptional.

Top recommendation

Deepen the A2A-visible stream adapter

The Session module now has depth; the next leakage is the seam where Session effects become A2A-visible stream responses. Deepening that adapter gives the most locality and test leverage without reopening ADR 0039.
